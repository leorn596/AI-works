## 2. 漏洞输入方式

平台支持三种输入方式，通过顶部 Tab 切换：

### 2.1 手动描述输入

![手动描述输入](screenshots/input-manual.png)

**适用场景：** 已知漏洞描述、安全测试发现、代码审计结果等。

**操作步骤：**

1. 切换到「手动描述」Tab
2. 在文本框中详细描述漏洞信息，包括：
   - 漏洞位置（URL、参数、端点）
   - 触发方式（输入 payload）
   - 响应特征（错误信息、异常行为）
   - 影响范围
3. 点击「开始 AI 分析」按钮
4. 等待 AI 完成分析（通常 10-30 秒）

**示例描述：**

```
在登录页面 /api/login 的 username 参数中输入单引号 '，
返回 MySQL 语法错误信息，怀疑存在 SQL 注入漏洞。
错误信息包含 "You have an error in your SQL syntax"。
```

> **提示：** 描述越详细，AI 分析结果越准确。最少 10 个字符。

### 2.2 文件上传（ZAP / Nmap）

![文件上传](screenshots/input-file.png)

**适用场景：** 已有 OWASP ZAP 或 Nmap 扫描报告，需要 AI 深度分析。

**操作步骤：**

1. 切换到「文件上传」Tab
2. 将扫描报告文件拖拽到上传区域，或点击选择文件
3. 支持的文件格式：
   - **ZAP JSON** — OWASP ZAP 导出的 JSON 格式报告
   - **Nmap XML** — Nmap 扫描的 XML 输出（`-oX` 参数）
4. 系统自动解析文件，显示解析结果（格式和漏洞数量）
5. 点击「开始 AI 批量分析」按钮
6. AI 将逐个分析每个漏洞并给出深度评估

![文件解析成功](screenshots/file-parsed.png)

> **注意：** 文件大小限制为 10MB。超过限制请先压缩或筛选关键漏洞。

### 2.3 多源交叉对比

![多源对比](screenshots/input-multi-source.png)

**适用场景：** 同时拥有 ZAP 和 Nmap 扫描结果，需要交叉验证。

**操作步骤：**

1. 切换到「多源对比」Tab
2. 分别上传 ZAP 和 Nmap 扫描报告
3. 系统执行交叉验证分析：
   - **匹配项** — 两个工具都发现的漏洞（高置信度）
   - **ZAP 独有** — 仅 ZAP 发现的漏洞
   - **Nmap 独有** — 仅 Nmap 发现的漏洞
   - **冲突项** — 两个工具结论不一致的漏洞
4. 查看置信度评分和匹配理由

## 3. 分析结果解读

分析完成后，仪表盘将展示以下内容：

![分析结果概览](screenshots/analysis-results.png)

### 3.1 摘要信息

- **综合 CVSS 评分** — 所有漏洞的综合严重程度评分（0-10）
- **漏洞数量统计** — 按类型分组的漏洞计数
- **AI 摘要** — 一段话总结整体安全状况

### 3.2 漏洞列表

每个漏洞条目包含：

| 字段 | 说明 |
|------|------|
| 漏洞名称 | 如「SQL Injection in Login」 |
| 漏洞类型 | SQLi / XSS / SSRF / CSRF / RCE 等 |
| CVSS 评分 | 0-10 分，越高越严重 |
| CVSS 向量 | 如 `CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H` |
| 描述 | 漏洞详细说明 |
| 修复建议 | 具体修复方案 |

### 3.3 漏洞严重等级

| CVSS 范围 | 等级 | 颜色标识 |
|-----------|------|---------|
| 9.0 - 10.0 | 🔴 严重 (Critical) | 红色 |
| 7.0 - 8.9 | 🟠 高危 (High) | 橙色 |
| 4.0 - 6.9 | 🟡 中危 (Medium) | 黄色 |
| 0.1 - 3.9 | 🟢 低危 (Low) | 绿色 |

## 4. 图表交互

平台提供四种可视化图表，支持交互式操作：

### 4.1 图表类型

![图表区域](screenshots/charts-area.png)

| 图表 | 展示内容 |
|------|---------|
| **饼图 (PieChart)** | 漏洞类型分布占比 |
| **柱状图 (BarChart)** | 各类型漏洞数量对比 |
| **雷达图 (RadarChart)** | 多维度安全评估（机密性/完整性/可用性等） |
| **趋势图 (TrendChart)** | 历史分析趋势变化 |

### 4.2 点击图表筛选

点击饼图或柱状图中的某个分类，将触发**跨图表联动筛选**：

1. 点击饼图中的「SQLi」扇区
2. 柱状图高亮 SQLi 对应的柱子
3. 雷达图更新为仅 SQLi 相关的维度数据
4. 漏洞列表过滤为仅显示 SQLi 类型漏洞

![图表联动](screenshots/chart-filter.png)

### 4.3 Event Bus 机制

图表联动基于**事件总线 (Event Bus)** 实现：

- 事件名称：`chart:filter`
- 事件数据：`{ type: 'SQLi', value: 'SQLi' }`
- 点击空白区域或再次点击可**取消筛选**

## 5. AI 深度分析

### 5.1 查看详细报告

在漏洞列表中，点击任意漏洞条目可展开详细 AI 分析：

![AI 详细分析](screenshots/ai-detail.png)

详细报告包含：

- **漏洞原理分析** — AI 对漏洞成因的技术解读
- **攻击向量** — 可能的攻击路径和利用方式
- **影响评估** — 对系统机密性、完整性、可用性的影响
- **修复建议** — 具体的代码级修复方案

### 5.2 加固清单

分析完成后自动生成**安全加固清单**：

![加固清单](screenshots/remediation-checklist.png)

- 按优先级排序（P1 最高，P5 最低）
- 按类别分组（配置 / 代码 / 网络 / 权限）
- 支持勾选标记已完成项
- 进度百分比实时更新

## 6. 历史记录

### 6.1 查看历史

通过顶部导航进入「历史记录」页面：

![历史记录](screenshots/history-list.png)

历史记录展示每次分析的：
- 任务 ID
- 输入类型（手动 / 文件 / 多源）
- 综合 CVSS 评分
- 漏洞数量
- 分析时间
- 状态

### 6.2 筛选和搜索

支持以下筛选条件：

| 筛选项 | 说明 |
|--------|------|
| 日期范围 | 选择开始和结束日期 |
| 漏洞类型 | 按 SQLi / XSS / SSRF 等类型过滤 |
| 分页 | 每页 20 条，支持翻页 |

### 6.3 回看详情

点击任意历史记录条目，可查看完整的分析结果，包括：
- 原始输入内容
- 所有漏洞详情
- 图表数据
- 加固清单（含完成状态）

## 7. 导出功能

### 7.1 导出 PNG 图表

![导出按钮](screenshots/export-buttons.png)

点击「导出 PNG」按钮，将所有图表导出为独立的 PNG 图片文件。

- 每个图表单独导出
- 自动命名（含任务 ID 和图表类型）
- 高分辨率，适合嵌入报告

### 7.2 导出 PDF 报告

点击「导出 PDF」按钮，可选择两种模式：

| 模式 | 内容 |
|------|------|
| **完整报告** | 概览 + 漏洞明细 + AI 分析 + 加固清单 |
| **摘要报告** | 仅概览信息（适合快速分享） |

PDF 报告包含：
- 封面（含生成时间和任务 ID）
- 综合评估摘要
- 漏洞详情表格
- AI 分析内容
- 加固清单

![PDF 报告示例](screenshots/pdf-preview.png)

## 8. 常见问题

### Q: 分析一直显示「加载中」怎么办？

检查后端服务是否正常运行：

```bash
curl http://localhost:8000/api/health
```

如果返回错误，请查看后端日志：`docker compose logs backend`

### Q: 文件上传后提示「未发现漏洞」？

请确认：
1. 文件格式是否为 ZAP JSON 或 Nmap XML
2. 文件中是否确实包含漏洞数据
3. 文件编码是否为 UTF-8

### Q: AI 分析结果不准确？

- 提供更详细的漏洞描述
- 包含具体的请求/响应数据
- 指明漏洞所在的 URL 和参数

### Q: 图表不显示或显示异常？

- 确认分析已完成（状态为「completed」）
- 刷新页面重试
- 检查浏览器控制台是否有报错

### Q: PDF 导出失败？

- 确认分析任务已完成
- 检查后端 PDF 服务是否正常
- 尝试切换为「摘要报告」模式
