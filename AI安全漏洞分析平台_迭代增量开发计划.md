# AI Web 安全漏洞报告与分析平台 — 迭代增量开发计划

**方法**：Scrum Sprint（固定时间盒，每 Sprint 2 周）  
**总周期**：8 个 Sprint / 16 周  
**前置文档**：`AI安全漏洞分析平台_项目策划书.md`

---

## Sprint 1：项目基建与前端骨架（第 1–2 周）

**目标**：搭通前后端基础链路，跑通"手动输入 → AI分析 → 结果展示"的最小闭环。

| ID | 故事点 | 描述 | 角色 |
|----|--------|------|------|
| US-01 | 5 | 搭建前端项目脚手架（React/Vue + 路由 + 状态管理） | 开发者 |
| US-02 | 3 | 搭建后端项目框架（FastAPI/Express + 路由结构） | 开发者 |
| US-03 | 3 | 创建MySQL核心表（`analysis_task` / `vulnerability`） | 开发者 |
| US-04 | 5 | 在输入区手动填写漏洞描述并提交分析 | 安全分析人员 |
| US-05 | 8 | AI接收手动输入并生成漏洞分析结果（基础版） | 安全分析人员 |
| US-06 | 5 | 提交分析后看到左侧漏洞列表和右侧AI分析区渲染 | 安全分析人员 |
| US-07 | 3 | 接入统一Loading状态（骨架屏/进度条） | 开发者 |
| US-08 | 2 | 配置CORS/环境变量/基础ErrorBoundary | 开发者 |

**Sprint 故事点**：34  
**交付物**：
- ✅ 前后端项目可运行
- ✅ 手动输入→AI分析→结果展示 全链路闭环
- ✅ MySQL表结构就绪
- ✅ 双栏布局显示（左：漏洞列表 / 右：AI分析区）
